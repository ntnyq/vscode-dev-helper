# AGENTS.md

This file helps AI coding agents work safely and quickly in this repository.

## Project Snapshot

- Type: VS Code extension (ESM, TypeScript)
- Runtime/framework: reactive-vscode
- Package manager: pnpm (locked to pnpm@11.3.0)
- Build tool: tsdown
- Main entry: src/index.ts

For feature and command reference, see [README.md](README.md).

## Install, Build, Validate

- Install dependencies: pnpm install --frozen-lockfile
- Dev watch build: pnpm run dev
- Production build: pnpm run build
- Lint: pnpm run lint
- Format check: pnpm run format:check
- Typecheck: pnpm run typecheck
- Full release checks: pnpm run release:check

## Local Debug Workflow

- Preferred loop:

1. Run pnpm run dev (watch mode)
2. Press F5 with the Run Extension launch config
3. Extension host opens using the playground workspace

Reference files:

- [.vscode/launch.json](.vscode/launch.json)
- [.vscode/tasks.json](.vscode/tasks.json)
- [playground/README.md](playground/README.md)

## Architecture Map

- [src/index.ts](src/index.ts): extension activation; wires command and provider registration.
- [src/commands/index.ts](src/commands/index.ts): command handlers and generation commands.
- [src/providers/codelenses/index.ts](src/providers/codelenses/index.ts): CodeLens provider registration.
- [src/providers/completions/index.ts](src/providers/completions/index.ts): completion provider registration.
- [src/providers/hovers/index.ts](src/providers/hovers/index.ts): hover provider registration.
- [src/templates/index.ts](src/templates/index.ts): generated file templates.
- [src/constants](src/constants): extension constants and language keys.
- [src/utils](src/utils): shared helpers and logger.

## Required Conventions

- Keep extension registration in src/index.ts simple and top-level.
- Prefer existing constants/utilities over new inline literals.
- Follow current formatting/linting via oxfmt and oxlint (do not introduce prettier/eslint-only workflows here).
- Keep feature code grouped by domain folder (commands/providers/templates/constants/utils).

## Generated File Rules

- [src/meta.ts](src/meta.ts) is generated by vscode-ext-gen. Do not hand-edit it.
- After command/config changes in [package.json](package.json), regenerate metadata with:
  - pnpm run generate:meta

Note: workspace settings run metadata generation on package.json save. See [.vscode/settings.json](.vscode/settings.json).

## Build/Runtime Gotchas

- tsdown copies oxidase WASM into dist at build time. If stripe types fail, verify tsdown copy config in [tsdown.config.ts](tsdown.config.ts).
- CI validates format, lint, build, and typecheck. Match that locally before release.
- There is no unit-test suite in this repo currently; rely on validation commands plus manual extension-host verification.

CI reference: [.github/workflows/ci.yml](.github/workflows/ci.yml)

## Change Playbooks

- Add or modify a command:

1. Update command implementation under [src/commands/index.ts](src/commands/index.ts) and helpers as needed.
2. Update contributes.commands/configuration in [package.json](package.json).
3. Run pnpm run generate:meta.
4. Run pnpm run lint && pnpm run typecheck && pnpm run build.

- Add or modify provider behavior:

1. Update provider class under [src/providers](src/providers).
2. Wire registration in the corresponding providers index file.
3. Validate in extension host with playground workspace.

## Release Notes

Release scripts are declared in [package.json](package.json), including version bump and publish flow. Use those scripts rather than ad hoc publish commands.
